# Why Simple Interpolation Fails

Our Round 1 submission used **Barycentric Interpolation** -- fitting a polynomial curve through the known context points. This has a fundamental flaw:

- It treats every signal identically --- a drum hit the same as a flute note
- It has no understanding of audio physics (harmonics, decay, periodicity)
- It can wildly overshoot in sparse regions

Real audio is **non-stationary**. A bass guitar has slow, periodic oscillations with gradual decay. A cymbal crash is high-frequency chaos. A synthesizer drone is perfectly periodic and harmonic. A good model must *infer which kind of signal it is looking at* before predicting the gaps.


# Our Round 2 Solution -- Transformer Neural Process

We train a **single Transformer model** on all 80,000 training samples. During training, it learns what different audio signals look like their shapes, periodicities, decay patterns, harmonic structures. At inference time, it uses this learned knowledge to fill gaps in any new sample in a **single forward pass**.

This approach is called **Amortized Inference** --- exactly what the problem brief requires. No per-sample optimization loops. One model, one pass, done.

# Why a Transformer?

Transformers use **self-attention** meaning every context point can look at every other context point simultaneously. This is crucial for audio because:

- A note at t=10ms provides information about what the signal will do at t=90ms
- The spacing between peaks tells you the frequency
- The rate of amplitude change tells you the decay envelope
- All of this is captured in a single attention operation

##  Cell-by-Cell Code Explanation

### Cell 1 -- Imports
Loads PyTorch, pandas, numpy, and checks GPU availability. .

### Cell 2 -- Load Data
Mounts Google Drive and reads the training CSV (~8 million rows, ~80,000 samples). Uses `pandas.read_csv`.

### Cell 3 -- Dataset Class (`AudioInpaintDataset`)
This is the data pipeline. For each `Sample_ID` it:
- Separates context rows (`Is_Context=1`) from target rows (`Is_Context=0`)
- Normalizes `Time_ms` to the range [0, 1] so the model sees consistent scale
- Pads or trims sequences to fixed lengths (`MAX_CONTEXT=50`, `MAX_TARGET=60`) so batches can be stacked on the GPU
- Returns a padding mask so the model knows which positions are real data vs padding

### Cell 4 -- Model (`AudioTransformerNP`)
Defines the Transformer Neural Process:
- `ctx_embed`: embeds each (time, value) pair into a 64-dimensional vector
- `tgt_embed`: embeds each target time into 64 dimensions
- `encoder`: 3-layer Transformer encoder with 4 attention heads; processes context points
- `decoder`: 3-layer Transformer decoder; cross-attends context memory to predict at target times
- `output_head`: linear layer projecting from 64 dimensions to a single predicted value

Total parameters: ~200,000 small enough to train quickly, powerful enough to learn audio structure.

### Cell 5 -- Training Loop
- Splits data 95/5 into train and validation sets
- Trains for 5 epochs with Adam optimizer (lr=1e-3)
- Loss =MSE only on real target points (padding is masked out)
- Gradient clipping (`max_norm=1.0`) prevents instability
- `ReduceLROnPlateau` halves the learning rate if validation loss stops improving
- Saves `best_model.pt` whenever validation MSE improves

### Cell 6 -- Inference (standard)
Loads the test CSV, runs the trained model, and writes `submission.csv`. Iterates sample by sample to correctly un-normalize predictions back to original time scale.

### Cell 8 — Blog / Summary
Prints a formatted explanation of the entire approach for the illustrated notebook requirement.
